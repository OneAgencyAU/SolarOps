Build a Microsoft 365 / Outlook OAuth connector for SolarOps. This mirrors the existing Google OAuth connector exactly. Use @azure/msal-node for the OAuth flow.

PACKAGE INSTALL:
npm install @azure/msal-node @microsoft/microsoft-graph-client

ENVIRONMENT VARIABLES (already in Replit secrets):
- MS_CLIENT_ID
- MS_CLIENT_SECRET  
- MS_TENANT_ID (value is "common")

BACKEND — create server/src/routes/microsoft.ts with these routes:

1. GET /api/auth/microsoft
   - Builds Microsoft OAuth URL using MSAL ConfidentialClientApplication
   - Config: { auth: { clientId: MS_CLIENT_ID, clientSecret: MS_CLIENT_SECRET, authority: `https://login.microsoftonline.com/${MS_TENANT_ID}` }}
   - Scopes: ['Mail.Read', 'Mail.ReadWrite', 'Mail.Send', 'User.Read', 'offline_access']
   - redirectUri: `${process.env.APP_URL || 'https://solarops.com.au'}/api/auth/microsoft/callback`
   - Store tenant_id and user_id in session before redirecting
   - Redirect user to the auth URL

2. GET /api/auth/microsoft/callback
   - Exchange code for tokens using MSAL acquireTokenByCode
   - Get user email by calling https://graph.microsoft.com/v1.0/me with the access token
   - Upsert into tenant_connections table: ms_access_token, ms_refresh_token, ms_email, ms_connected_at = now(), ms_last_sync = now()
   - Redirect to /connections?ms_connected=true on success, /connections?error=ms_auth_failed on failure

3. GET /api/auth/microsoft/status?tenant_id=X
   - Query tenant_connections for ms_email and ms_connected_at
   - Return { connected: boolean, email: string|null, lastSync: string|null }

4. DELETE /api/auth/microsoft/disconnect?tenant_id=X
   - Clear ms_access_token, ms_refresh_token, ms_email, ms_connected_at, ms_last_sync for the tenant
   - Return { success: true }

5. GET /api/microsoft/emails?tenant_id=X&folder=inbox&limit=20
   - Fetch ms_access_token from tenant_connections
   - Call https://graph.microsoft.com/v1.0/me/messages?$top=20&$orderby=receivedDateTime desc&$select=id,subject,from,receivedDateTime,bodyPreview,isRead,conversationId
   - Return formatted emails matching the same shape as the Google emails endpoint: { id, threadId, subject, from: { name, email }, date, snippet, isRead, labelIds }

6. GET /api/microsoft/email/:id?tenant_id=X
   - Fetch full email body from https://graph.microsoft.com/v1.0/me/messages/{id}
   - Return { id, subject, from, date, body, headers }

7. POST /api/microsoft/draft
   - Body: { tenant_id, to, subject, body, replyToMessageId? }
   - If replyToMessageId provided, use reply endpoint: POST /v1.0/me/messages/{id}/createReply
   - Otherwise create draft: POST /v1.0/me/messages with isDraft: true
   - Return { draftId, success }

8. POST /api/microsoft/send
   - Body: { tenant_id, draftId }
   - Call POST /v1.0/me/messages/{draftId}/send
   - Update ms_last_sync in tenant_connections
   - Return { success }

Register the router in server/src/index.ts:
import microsoftRouter from './routes/microsoft';
app.use(microsoftRouter);
