Install required packages:
- @google-cloud/vision (Google Cloud Vision API)
- multer (file upload handling)

ENDPOINT 1: POST /api/bill-reader/check
- Accepts: multipart form with file (image or PDF)
- Purpose: quick pre-check before running OCR
- Process:
  1. Get the filename and file mimetype
  2. If it's an image, use Google Vision API's 
     TEXT_DETECTION with max 1 page to get a small 
     sample of text (first 500 chars only)
  3. If it's a PDF, use Google Vision API to get 
     first page text sample only
  4. Send sample text + filename to Claude API 
     (claude-haiku-3-5 for speed and cost) with prompt:
     "You are a document classifier. Based on this 
     filename and text sample, determine if this is 
     an Australian electricity bill. 
     Respond with JSON only:
     {
       'isBill': true/false,
       'confidence': 0.0-1.0,
       'reason': 'brief explanation'
     }
     Filename: [filename]
     Text sample: [first 500 chars]"
  5. Return the classification result
- Use GOOGLE_VISION_API_KEY and ANTHROPIC_API_KEY env vars

ENDPOINT 2: POST /api/bill-reader/extract
- Accepts: multipart form with file (image or PDF)
- Purpose: full extraction after pre-check passes
- Process:
  1. Send full file to Google Vision API TEXT_DETECTION
  2. Get complete extracted text
  3. Send to Claude API (claude-sonnet-4) with prompt:
     "You are an Australian electricity bill parser.
     Extract the following fields from this bill text.
     If a field is not found, return null.
     Respond with JSON only, no other text:
     {
       'nmi': string | null,
       'retailer': string | null,
       'customerName': string | null,
       'propertyAddress': string | null,
       'billingPeriod': {
         'from': string | null,
         'to': string | null,
         'days': number | null
       },
       'usage': {
         'dailyAvgKwh': number | null,
         'totalKwh': number | null,
         'peakKwh': number | null,
         'offPeakKwh': number | null,
         'shoulderKwh': number | null
       },
       'rates': {
         'supplyCharge': number | null,
         'usageRate': number | null,
         'peakRate': number | null,
         'offPeakRate': number | null,
         'feedInTariff': number | null
       },
       'totals': {
         'totalAmount': number | null,
         'gstAmount': number | null
       },
       'existingSolar': boolean | null,
       'existingBattery': boolean | null,
       'meterType': string | null,
       'meterCondition': string | null,
       'confidenceScore': number
     }
     Bill text: [full OCR text]"
  4. Return structured JSON + raw OCR text

ENDPOINT 3: POST /api/bill-reader/save
- Accepts: extracted bill data JSON + optional jobId
- Saves to Supabase bill_extractions table
- Returns saved record with ID

Create Supabase migration:

  supply_charge NUMERIC,
  usage_rate NUMERIC,
  feed_in_tariff NUMERIC,

  existing_solar BOOLEAN,
