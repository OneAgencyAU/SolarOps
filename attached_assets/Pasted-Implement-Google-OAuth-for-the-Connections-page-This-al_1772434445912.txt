Implement Google OAuth for the Connections page. This allows 
users to connect their Google Workspace (Gmail) account to 
SolarOps. Do not change any other pages or functionality.

BACKEND CHANGES:

1. Install required packages:
   - passport
   - passport-google-oauth20
   - express-session
   - connect-pg-simple (for session storage in Supabase)

2. Create server/src/config/passport.ts:
   - Configure passport with GoogleStrategy
   - Use GOOGLE_CLIENT_ID and GOOGLE_CLIENT_SECRET env vars
   - Callback URL: /api/auth/google/callback
   - Scopes: profile, email, 
     https://www.googleapis.com/auth/gmail.readonly,
     https://www.googleapis.com/auth/gmail.compose
   - On success: store in Supabase google_connections table
     (tenant_id, user_id, google_email, access_token, 
     refresh_token, scopes, connected_at)
   - Serialize/deserialize user by Firebase UID

3. Create Supabase migration for google_connections table:
   CREATE TABLE google_connections (
     id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
     tenant_id UUID REFERENCES tenants(id) ON DELETE CASCADE,
     user_id TEXT REFERENCES users(id) ON DELETE CASCADE,
     google_email TEXT NOT NULL,
     access_token TEXT,
     refresh_token TEXT,
     scopes TEXT[],
     connected_at TIMESTAMPTZ DEFAULT NOW(),
     last_sync TIMESTAMPTZ DEFAULT NOW()
   );
   ALTER TABLE google_connections ENABLE ROW LEVEL SECURITY;

4. Create server/src/routes/auth.ts with these endpoints:
   GET /api/auth/google — initiates OAuth flow
   GET /api/auth/google/callback — handles callback, 
     stores tokens, redirects to /connections?connected=true
   GET /api/auth/google/status — returns connection status 
     for current tenant
     Response: { connected: boolean, email: string | null, 
     lastSync: string | null }
   DELETE /api/auth/google/disconnect — removes connection 
     from google_connections table

5. Add express-session middleware to server/src/index.ts:
   - secret: SESSION_SECRET env var
   - resave: false
   - saveUninitialized: false
   - cookie: { secure: false, maxAge: 24 * 60 * 60 * 1000 }

6. Register passport and auth routes in server/src/index.ts
